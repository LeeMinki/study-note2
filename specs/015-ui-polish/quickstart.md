# Quickstart: UI Polish

**Feature**: 015-ui-polish
**Date**: 2026-04-21

개발 환경에서 UI 개선 결과를 직접 확인하는 방법과 시나리오별 검증 체크리스트.

## 개발 서버 실행

```bash
# 백엔드 (필요 시)
cd backend && JWT_SECRET=dev-secret-min-32-chars-long npm start

# 프론트엔드
cd frontend && npm run dev
# → http://localhost:5173
```

## 시나리오별 검증

### S1 — 인증 화면 (AuthForm)

1. 브라우저에서 http://localhost:5173 접근 (미인증 상태)
2. **확인 항목**:
   - [ ] 페이지 중앙 수직 정렬: 로그인 폼이 화면 세로 중앙 근처에 위치
   - [ ] 브랜드 영역: "Study Note" eyebrow + "Capture fast. Find faster." h1 + heroText 영어 설명 표시
   - [ ] 탭: "로그인" / "회원가입" 전환 동작 정상
   - [ ] Google 로그인 버튼: 로그인 탭에서만 표시, `ssoButton` 스타일 정상
   - [ ] 모바일(320px): 개발자 도구에서 320px 너비 설정 후 폼 잘림 없음
   - [ ] 오류 표시: 잘못된 비밀번호 입력 후 `errorText` 스타일로 폼 내부 표시

### S2 — 메인 화면 헤더

1. 로그인 완료 후 메인 화면
2. **확인 항목**:
   - [ ] Hero h1: "안녕하세요, [표시이름]님" 형태로 사용자 이름 표시
   - [ ] heroText: "노트를 작성하고 검색하세요." 표시
   - [ ] 프로필 버튼 + 로그아웃 버튼: 우측 상단에 명확히 구별
   - [ ] 검색바, 태그 필터: 정상 동작
   - [ ] 에러 발생 시: `errorBanner` 스타일로 표시

### S3 — 노트 목록 상태

1. 로그인 완료 후 노트 로딩 시
2. **확인 항목**:
   - [ ] 로딩 중: CSS 스피너(회전 원)와 "노트 불러오는 중…" 텍스트가 함께 표시
   - [ ] 빈 상태: "아직 노트가 없습니다" + "첫 번째 노트를 작성해보세요." 한국어 표시
   - [ ] 필터 결과 없음: "일치하는 노트가 없습니다" + "필터 초기화" 버튼
   - [ ] "필터 초기화" 클릭 시 검색어/태그 초기화 동작

### S4 — 노트 카드

1. 노트 1개 이상 있는 상태
2. **확인 항목**:
   - [ ] 카드 기본 뷰: 제목, 날짜, 본문 미리보기, 태그, "편집" / "삭제" 버튼
   - [ ] "편집" 클릭 → 편집 모드: "저장" / "취소" 버튼, 입력 필드
   - [ ] 제목 60자+ 노트: 카드 너비 내에서 줄바꿈 처리, 레이아웃 깨짐 없음
   - [ ] 태그 10개+ 노트: 태그 목록이 wrap되며 카드 너비 유지
   - [ ] 편집 모드에서 제목 비운 채 저장: "제목은 필수입니다." 한국어 오류 표시
   - [ ] 마크다운 본문 (코드블록, 목록): 카드 너비 내에서 렌더링

### S5 — 노트 작성 (NoteComposer)

1. 메인 화면 좌측 패널
2. **확인 항목**:
   - [ ] 섹션 제목: "새 노트"
   - [ ] 부제목: "Ctrl/Cmd + Enter로 빠르게 저장"
   - [ ] 제목 placeholder: "제목"
   - [ ] 내용 placeholder: "내용 (마크다운 지원)"
   - [ ] 태그 placeholder: "태그 (쉼표로 구분)"
   - [ ] 저장 버튼: "저장"
   - [ ] 제목 없이 저장: "제목은 필수입니다." 한국어 표시
   - [ ] 드래프트 배너: "이전에 작성 중이던 내용이 있습니다." (기존 한국어 유지)

### S6 — 프로필 화면

1. 우상단 "프로필" 버튼 클릭
2. **확인 항목**:
   - [ ] 계정 요약, 프로필 수정, 비밀번호 변경 섹션이 시각적으로 구분
   - [ ] Google 연결 섹션: 연결 버튼 또는 "연결됨" 상태 표시
   - [ ] Google 연결 성공 후 리다이렉트: 성공 메시지 `.successText` (초록색) 표시
   - [ ] 오류 발생 시: `.errorText` (빨간색) 표시
   - [ ] "뒤로" 또는 이전 화면으로 이동 버튼 동작

### S7 — 반응형

브라우저 개발자 도구에서 뷰포트 너비 조절:

1. **1400px**: 기본 2열 레이아웃 (NoteComposer + NoteList)
2. **880px**: 경계선 — 1열 전환 시작 확인
3. **600px**: 1열, 모든 요소 수직 배치
4. **320px**:
   - [ ] 텍스트 잘림 없음
   - [ ] 버튼 잘림 없음
   - [ ] 입력 필드 잘림 없음
   - [ ] 스크롤 없이 인증 폼 접근 가능

### S8 — CSS 디자인 토큰 검증

```bash
# CSS 변수가 올바르게 정의되어 있는지 확인
grep -n "var(--" frontend/src/styles/app.css | head -20

# :root 블록에 변수 정의 확인
grep -n "^\s*--" frontend/src/styles/app.css
```
