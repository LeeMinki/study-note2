# Feature Specification: 이미지 붙여넣기 및 자동 임시저장

**Feature Branch**: `003-image-paste-autosave`
**Created**: 2026-04-14
**Status**: Draft
**Input**: Jira SP-18 — "글 내용에 사진을 넣고 싶음. 사진 ctrl+c/v 누르면 알아서 마크다운 링크가 들어가는 기능. 저장하면 렌더링된 곳에서 사진이 보임. 글 내용이 길어지다보면 임시저장이 꼭 필요함."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 이미지 붙여넣기 → 마크다운 자동 삽입 (Priority: P1)

사용자가 노트 내용 편집 중 클립보드에 복사한 이미지를 Ctrl+V로 붙여넣으면, 이미지가 자동으로 업로드되고 마크다운 이미지 문법(`![image](url)`)이 커서 위치에 삽입된다.

**Why this priority**: 사용자가 가장 직접적으로 원하는 기능이며, 이미지 렌더링(P2)의 전제가 된다.

**Independent Test**: 클립보드에 이미지를 복사한 뒤 content 편집 영역에서 Ctrl+V를 누르면, 텍스트 영역에 `![image](...)` 형태의 마크다운이 삽입되고 업로드 완료 확인이 가능하다.

**Acceptance Scenarios**:

1. **Given** 사용자가 클립보드에 이미지를 복사한 상태에서 content textarea에 포커스가 있을 때, **When** Ctrl+V를 누르면, **Then** 이미지가 백엔드에 업로드되고 `![image](업로드된_url)` 문자열이 커서 위치에 삽입된다.
2. **Given** 이미지 업로드가 진행 중일 때, **When** 업로드가 완료되기 전이라면, **Then** 편집 영역에 업로드 진행 상태를 나타내는 임시 텍스트(예: `![업로드 중...]`)가 표시되다가 완료 후 실제 URL로 교체된다.
3. **Given** 사용자가 텍스트(이미지 아님)를 클립보드에 복사한 상태에서 Ctrl+V를 누를 때, **When** 붙여넣기가 실행되면, **Then** 기존 텍스트 붙여넣기 동작이 그대로 유지된다.
4. **Given** 이미지 업로드가 실패했을 때, **When** 오류가 발생하면, **Then** 임시 텍스트가 제거되고 사용자에게 업로드 실패 메시지가 표시된다.

---

### User Story 2 - 렌더링 영역에서 이미지 표시 (Priority: P2)

저장된 노트의 마크다운 렌더링 영역에서 `![alt](url)` 문법이 실제 이미지로 표시된다.

**Why this priority**: P1에서 삽입된 이미지 링크가 실제로 보여야 기능이 완성된다. P1 없이도 기존 마크다운 이미지 링크를 렌더링하는 독립 기능이다.

**Independent Test**: 노트 content에 `![test](http://...)` 형태의 마크다운이 포함된 상태로 저장 후 렌더링 영역을 확인하면 이미지가 표시된다.

**Acceptance Scenarios**:

1. **Given** 노트 content에 `![alt](url)` 형태의 마크다운이 포함된 노트가 저장되어 있을 때, **When** 노트 카드의 렌더링 영역을 보면, **Then** 해당 위치에 이미지가 표시된다.
2. **Given** 이미지 URL이 깨지거나 로드에 실패할 때, **When** 렌더링 영역에서 해당 이미지를 표시하려 하면, **Then** 이미지 대신 alt 텍스트 또는 대체 표시가 나타난다.
3. **Given** 렌더링 영역에 이미지가 포함될 때, **When** 이미지 크기가 컨테이너보다 클 경우, **Then** 이미지가 컨테이너 너비를 초과하지 않도록 조정된다.

---

### User Story 3 - 작성 중 자동 임시저장 및 복원 (Priority: P3)

노트 작성 중 내용이 일정 시간 후 자동으로 임시저장되어, 실수로 페이지를 벗어나거나 브라우저를 닫아도 작성 중이던 내용이 복원된다.

**Why this priority**: 긴 글 작성 시 내용 손실을 방지하는 안전망. P1/P2와 독립적으로 동작한다.

**Independent Test**: 내용을 입력한 뒤 저장하지 않고 페이지를 새로고침하면 이전에 작성 중이던 내용이 자동으로 복원된다.

**Acceptance Scenarios**:

1. **Given** 사용자가 NoteComposer에 내용을 입력하고 3초 이상 멈췄을 때, **When** 자동저장 인터벌이 트리거되면, **Then** 현재 입력 중인 제목/내용/태그가 브라우저에 임시저장된다.
2. **Given** 임시저장된 데이터가 있는 상태에서 페이지를 새로고침했을 때, **When** NoteComposer가 초기화되면, **Then** 이전에 임시저장된 내용이 폼에 자동으로 복원되고 복원 안내 메시지가 표시된다.
3. **Given** 사용자가 노트를 성공적으로 저장했을 때, **When** 저장이 완료되면, **Then** 해당 임시저장 데이터가 삭제되어 다음 작성 시 빈 폼으로 시작한다.
4. **Given** 사용자가 임시저장된 내용을 원하지 않을 때, **When** 복원 안내에서 "삭제" 또는 "무시"를 선택하면, **Then** 임시저장 데이터가 삭제되고 빈 폼이 표시된다.

---

### Edge Cases

- 클립보드에 이미지와 텍스트가 함께 있을 경우 어떻게 처리하는가? (가정: 이미지 우선 처리)
- 매우 큰 이미지(예: 10MB 이상) 붙여넣기 시 어떻게 처리하는가? (가정: 최대 파일 크기 제한 + 오류 메시지)
- 임시저장 데이터와 실제 노트가 동시에 존재할 때 충돌 방지는?
- 여러 탭에서 동시에 노트를 작성할 경우 임시저장 데이터 간 충돌 방지는?
- 오프라인 상태에서 이미지를 붙여넣으면 어떻게 처리하는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 content 편집 영역에서 Ctrl+V(붙여넣기) 이벤트 발생 시 클립보드 데이터가 이미지인지 감지해야 한다.
- **FR-002**: 이미지가 감지되면 시스템은 해당 이미지를 백엔드에 업로드하고 반환된 URL로 마크다운 이미지 문법을 생성하여 커서 위치에 삽입해야 한다.
- **FR-003**: 이미지 업로드 중에는 편집 영역에 진행 상태를 나타내는 임시 텍스트가 표시되어야 한다.
- **FR-004**: 이미지 업로드 실패 시 사용자에게 오류 메시지를 표시하고 임시 텍스트를 제거해야 한다.
- **FR-005**: 노트 렌더링 영역은 마크다운 `![alt](url)` 문법을 실제 `<img>` 요소로 렌더링해야 한다.
- **FR-006**: 렌더링된 이미지는 컨테이너 너비를 초과하지 않도록 최대 너비가 제한되어야 한다.
- **FR-007**: 이미지 로드 실패 시 렌더링 영역에 alt 텍스트 또는 대체 표시가 나타나야 한다.
- **FR-008**: 시스템은 NoteComposer 입력 내용 변경 후 3초 이내에 브라우저에 임시저장해야 한다.
- **FR-009**: 임시저장된 데이터가 있는 경우 페이지 로드 시 NoteComposer에 자동 복원하고 사용자에게 복원 안내를 표시해야 한다.
- **FR-010**: 노트 저장 성공 시 해당 임시저장 데이터를 자동으로 삭제해야 한다.
- **FR-011**: 사용자가 임시저장 데이터 삭제를 요청하면 데이터를 삭제하고 빈 폼을 표시해야 한다.

### Constitution Alignment *(mandatory)*

- **CA-001**: 이미지 파일 업로드 및 저장은 `/backend`에서 처리한다. 프론트엔드는 HTTP API를 통해서만 업로드 요청을 전송하며 파일 시스템에 직접 접근하지 않는다.
- **CA-002**: 이미지 업로드 엔드포인트 `POST /api/images` 신규 추가. 응답은 `{ success: boolean, data: { url: string }, error: string | null }` 형식. 업로드된 이미지는 백엔드 로컬 디렉토리(`/uploads`)에 저장.
- **CA-003**: 이미지 업로드에 `multer` 패키지가 필요하며 **사용자 승인 필요**. 프론트엔드는 기존 fetch API로 충분하여 신규 의존성 불필요. 마크다운 렌더러의 이미지 지원은 기존 렌더러 확장으로 처리.
- **CA-004**: 이미지 붙여넣기는 content textarea의 paste 이벤트 핸들러로 인라인 처리. 별도 모달 없음. 임시저장 복원 안내는 NoteComposer 내 인라인 배너로 표시.
- **CA-005**: 이미지 메타데이터(파일명, URL, 크기)는 백엔드 JSON 파일 또는 별도 uploads 디렉토리 인덱스에 저장. 임시저장 데이터는 localStorage에만 저장.

### Key Entities *(include if feature involves data)*

- **UploadedImage**: 파일명, 저장 경로, URL, 업로드 시각, 파일 크기
- **DraftNote**: 임시저장된 노트 폼 상태 — 제목, 내용, 태그, 마지막 저장 시각 (localStorage)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Ctrl+V 붙여넣기부터 마크다운 삽입 완료까지 네트워크 정상 조건에서 3초 이내에 완료된다.
- **SC-002**: 이미지 붙여넣기 시 기존 텍스트 붙여넣기 동작이 100% 유지된다 (이미지가 아닌 경우).
- **SC-003**: 렌더링 영역의 마크다운 이미지가 컨테이너 너비를 초과하지 않는다.
- **SC-004**: 입력 변경 후 3초 이내에 임시저장이 완료된다.
- **SC-005**: 페이지 새로고침 후 임시저장 데이터 복원 성공률 100%.
- **SC-006**: 노트 저장 성공 시 임시저장 데이터가 자동 삭제되어 다음 작성 시 빈 폼이 표시된다.

## Assumptions

- 이미지 저장은 백엔드 로컬 파일시스템(`/uploads` 디렉토리)에 저장한다. 외부 스토리지 서비스(S3 등)는 MVP 범위 밖이다.
- 이미지 업로드에 `multer` 패키지 사용이 필요하며 사용자 승인 후 설치한다.
- 최대 업로드 파일 크기는 5MB로 제한한다.
- 지원 이미지 형식: PNG, JPEG, GIF, WebP.
- 임시저장은 새 노트 작성 폼(NoteComposer)에만 적용된다. 기존 노트 편집(NoteCard 인라인 편집)은 MVP 범위 밖이다.
- 자동저장 트리거: 입력 변경 후 3초 debounce (타이머 기반).
- 임시저장 키: `study-note-draft` (localStorage).
- 이미지 URL은 백엔드 서빙 경로(`/uploads/filename`)로 생성된다.
- 클립보드에 이미지와 텍스트가 함께 있을 경우 이미지를 우선 처리한다.
